# 📌 React Lifecycle & useEffect 정리

## 📖 개요

React 컴포넌트의 라이프사이클과 useEffect를 활용한 제어 방법 정리

------------------------------------------------------------------------

## 🧠 핵심 개념

-   컴포넌트는 생성 → 업데이트 → 제거 과정을 거침
-   useEffect를 통해 라이프사이클 제어 가능

------------------------------------------------------------------------

## 🔄 라이프사이클

1.  **Mount (마운트)**
    -   컴포넌트가 처음 렌더링됨
2.  **Update (업데이트)**
    -   state / props 변경 시 리렌더링
3.  **Unmount (언마운트)**
    -   컴포넌트가 화면에서 제거됨

------------------------------------------------------------------------

## ⚙️ useEffect 기본

``` jsx
useEffect(() => {
  console.log("업데이트");
});
```

-   의존성 배열 없으면 → 렌더링마다 실행

------------------------------------------------------------------------

## 🎯 특정 값 감지

``` jsx
useEffect(() => {
  console.log("변경 감지:", count, text);
}, [count, text]);
```

-   배열 안 값이 바뀔 때만 실행됨

------------------------------------------------------------------------

## 🧩 마운트 제어

``` jsx
useEffect(() => {
  console.log("마운트 시 실행");
}, []);
```

-   빈 배열 → 최초 1회 실행

------------------------------------------------------------------------

## 🔥 업데이트만 감지 (마운트 제외)

``` jsx
import { useRef, useEffect } from "react";

const didMountRef = useRef(false);

useEffect(() => {
  if (!didMountRef.current) {
    didMountRef.current = true;
    return;
  }
  console.log("업데이트!");
});
```

------------------------------------------------------------------------

## 🧹 클린업 (Cleanup)

``` jsx
useEffect(() => {
  const id = setInterval(() => {
    console.log("실행");
  }, 1000);

  return () => {
    clearInterval(id);
  };
});
```

-   이전 effect 정리
-   메모리 누수 방지

------------------------------------------------------------------------

## ❗ 클린업 필요 이유

❌ 잘못된 코드

``` jsx
useEffect(() => {
  setInterval(() => {
    console.log("깜빡");
  }, 1000);
});
```

👉 문제 - 렌더링마다 interval 생성 - 중복 실행 발생

------------------------------------------------------------------------

## ✅ 올바른 코드

-   cleanup으로 interval 제거

------------------------------------------------------------------------

## 🧠 언마운트 제어

``` jsx
useEffect(() => {
  return () => {
    console.log("언마운트");
  };
}, []);
```

-   컴포넌트 제거 시 실행됨

------------------------------------------------------------------------

## 🎯 조건부 렌더링 + 언마운트

``` jsx
{count % 2 === 0 && <Even />}
```

👉 흐름

-   짝수 → 컴포넌트 생성 (Mount)
-   홀수 → 컴포넌트 제거 (Unmount)
-   이때 cleanup 실행

------------------------------------------------------------------------

## 📌 핵심 정리

-   useEffect = 사이드 이펙트 처리 도구
-   의존성 배열로 실행 시점 제어
-   cleanup 함수는 언마운트 또는 재실행 전에 실행됨

------------------------------------------------------------------------
